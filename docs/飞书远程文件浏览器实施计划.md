# 飞书远程文件浏览器实施计划

## 1. 实施结果

本计划落实 `docs/飞书远程文件浏览器方案设计.md`。当前版本已完成只读远程目录浏览、文本文件分页、Card JSON 2.0 交互、回调安全和可靠性治理。

实施原则：

- 文件只通过 OpenCode Server API 读取，不访问桥接主机项目文件系统。
- 浏览操作不调用模型，不产生模型 Token。
- callback 不信任路径、session ID、directory 或页码。
- 同卡操作串行执行，重复和旧视图回调在同步阶段拒绝。
- 第一阶段不包含编辑、远程下载、搜索或状态持久化。

## 3. 已完成实现

### 3.1 OpenCode 文件访问

- [x] 使用 `@opencode-ai/sdk` v2 `session.get()` 获取绑定 session 的 `directory`。
- [x] 使用 `file.list()` 读取当前目录直接子项。
- [x] 使用 `file.read()` 读取文本文件。
- [x] 过滤 OpenCode 标记为 ignored 的目录项。
- [x] 二进制文件返回不可预览错误。
- [x] 文本文件设置 1 MiB 预览上限。
- [x] session 和目录请求设置 5 秒超时。
- [x] 文件请求设置 10 秒超时。
- [x] 使用 `AbortSignal` 中止超时请求。
- [x] 归一化 not found、forbidden、binary、too large、timeout、invalid response 和 unavailable 错误。

### 3.2 路径与敏感文件策略

- [x] 仅接受项目内相对路径。
- [x] 拒绝绝对路径、空字节、反斜杠和 `..` 逃逸。
- [x] 集中实现路径规范化、父目录和子项拼接。
- [x] 默认隐藏并拒绝 `.env`、`.env.*`、私钥和证书文件。
- [x] callback 不接受客户端传入的真实路径、directory 或 session ID。
- [x] 不使用桥接主机 `fs.realpath()` 冒充远程文件校验。

### 3.3 目录卡片

- [x] 使用 Card JSON 2.0 和 `width_mode: fill`。
- [x] 完整展示 session ID，不做固定长度截断。
- [x] 路径以 `/src/handler` 形式展示。
- [x] 目录优先、名称不区分大小写排序。
- [x] 目录每页显示 8 项。
- [x] 目录使用 `📁`，文件使用 `📄`。
- [x] 不显示 `[目录]`、`[文件]` 文本前缀。
- [x] 每项为单列全宽文本按钮，图标和名称均可点击。
- [x] 不显示右侧箭头。
- [x] 上一级、根目录、刷新、上一页和下一页合并到一个 `flex_mode: none` 单行操作区。
- [x] 边界上不存在的操作不渲染，其余按钮保持单行。
- [x] 空目录显示明确空状态。

### 3.4 文件预览卡

- [x] 文本文件每页最多 60 行。
- [x] 单页代码内容限制约 14 KiB。
- [x] 展示文件路径、当前行范围和总行数。
- [x] 根据扩展名选择代码围栏语言。
- [x] 中和文件内容中的三反引号，避免破坏卡片 Markdown。
- [x] 支持返回目录、刷新、上一页和下一页。
- [x] 文件页所有操作固定在同一个单行操作区。
- [x] 文件缩短后自动修正到有效页码。

### 3.5 服务端浏览状态

- [x] 以飞书卡片 message ID 注册浏览器状态。
- [x] 保存 chat ID、创建者 open ID、session ID 和 workspace directory。
- [x] 保存目录或文件视图、返回目录和返回页。
- [x] 每次渲染生成随机 `viewToken`。
- [x] entry key 绑定当前 view token 和服务端 entry map。
- [x] 每个动作携带稳定 `actionId`。
- [x] 入队前将 entry key 解析为不可变目标动作。
- [x] 重复 callback 返回“该操作正在处理中”。
- [x] 旧视图 callback 返回“目录已更新，请点击最新卡片”。
- [x] 非创建者操作返回明确无权限 toast。
- [x] 浏览状态默认 TTL 为 30 分钟。
- [x] 合法操作后续期，服务关闭时清理所有 timer。

### 3.6 回调与可靠性

- [x] WebSocket 和 HTTP webhook 均支持文件浏览同步预检。
- [x] 在飞书 3 秒预算内完成 message、chat、operator、viewToken、entryKey 和 actionId 校验。
- [x] 合法动作立即返回“正在打开目录/文件”等准确 toast。
- [x] 过期、无权限、重复和旧视图返回不同 toast。
- [x] 合法动作显示轻量加载卡后请求 OpenCode。
- [x] 每张卡片使用独立 Promise 队列。
- [x] 单次失败不会永久破坏后续队列。
- [x] 飞书 `updateMessage()` 最多尝试 3 次，短退避为 200ms、600ms。
- [x] 加载态更新失败不阻塞实际目录或文件请求。
- [x] 远端错误更新为可恢复错误卡，保留根目录和重试。

## 6. 后续候选能力

以下能力不属于当前版本：

- 按文件名或内容搜索。
- 面包屑分段点击。
- 跳转到指定行。
- 最近访问和收藏路径。
- 远端文件下载和图片预览。
- Git 状态和 diff 展示。
- 文件编辑。
- 浏览状态持久化和服务重启恢复。

这些能力必须复用当前 `RemoteFileClient`、路径策略、view token 和回调鉴权模型，并单独补充安全设计。
