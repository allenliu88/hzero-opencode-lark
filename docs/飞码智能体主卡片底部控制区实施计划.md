## 3. 已完成 API 契约

- [x] Agent：`GET /agent?directory=...`。
- [x] Provider/Model：`GET /provider?directory=...`。
- [x] Session：`GET /session?directory=...&roots=true&limit=10000`。
- [x] Session 信息和当前模型：`GET /session/:sessionID`。
- [x] 分支：`GET /vcs?directory=...`。
- [x] `OpencodeControlClient.abortSession()` 支持项目 scoped abort 失败时 fallback `/session/:sessionID/abort`；输入 `/abort` 当前直接调用全局 endpoint。
- [x] 消息 body 支持 agent/provider/model，失败时 fallback `{ parts }`。

## 4. 已完成数据层

- [x] mapping 持久化 `project_id`、`directory`、`provider_id`、`model_id`。
- [x] mapping 持久化 `session_title`、`project_name`、`branch_name`。
- [x] `SessionManager.updateContext()` 仅更新指定字段。
- [x] `SessionOption` 解析 title、directory、summary、time 和 model。
- [x] Session API 模型归一化为 `providerId/modelId`。

## 5. 已完成选择器

- [x] `/agents`、`/models`、`/sessions` 使用同一 registry。

- [ ] 在主卡渲染直接打开 Agent、Model、Session 选择器的按钮；当前主卡只展示帮助命令。
- [x] Card JSON 2.0 全宽按钮。
- [x] 每页 8 行、上一页/下一页、原卡更新和页码。

- [x] Model 排除 deprecated；响应提供 `connected` 时只展示 connected provider，缺失时兼容全部可解析 provider。
- [x] Model 只显示 `providerID/modelID`。
- [x] Session 当前项置顶，显示名称、相对时间、文件数和灰色 ID/项目分支。

## 11. 回滚策略

- 不传 `AgentConsoleOptions.controls` 可隐藏主卡 footer。
- 不注册 `SelectionPickerRegistry` 可关闭三类选择卡入口。
- SQLite 新增列无需回滚，旧代码可忽略。
- OpenCode agent/model 字段不兼容时，消息发送自动回退为仅发送 parts。
